## Claude Code 记忆系统调研

### 时间：晚间 23:13

调研了 `claude-code-rev` 项目的记忆系统实现，核心发现如下：

---

### 一、整体架构：三层分离

```
┌─────────────────────────────────────────────────────────────┐
│                     CC记忆系统架构                           │
├─────────────────────────────────────────────────────────────┤
│  memdir/        │  SessionMemory/    │  autoDream/          │
│  持久记忆层      │  会话记忆层         │  整合层(做梦)        │
│  ~/.claude/     │  /tmp/session.md   │  forkedAgent        │
│  memory/        │  (动态笔记)         │  历史会话整合        │
│  MEMORY.md      │                    │                     │
│  topic.md       │                    │                     │
├─────────────────────────────────────────────────────────────┤
│  extractMemories - 对话结束时提取 → 写入持久记忆             │
└─────────────────────────────────────────────────────────────┘
```

---

### 二、四种记忆类型

| 类型    | 内容                  | 范围           |
|---------|----------------------|----------------|
| user    | 用户角色、偏好、知识   | 私有           |
| feedback| 行为指导（纠正、确认） | 默认私有        |
| project | 项目状态、决策、进展   | 默认team        |
| reference| 外部资源指针          | 默认team        |

---

### 三、关键设计哲学

#### 1. 排除可推导内容（核心）

```
❌ 不存: 代码模式、架构、文件路径、项目结构
❌ 不存: Git历史、recent changes
❌ 不存: 调试方案、修复配方
❌ 不存: CLAUDE.md已有内容

✓ 只存: 无法从当前状态推导的上下文
```

#### 2. 索引与内容分离

- MEMORY.md = 索引（200行上限）
- topic.md = 详细内容（带frontmatter）

#### 3. 自然断点触发

- SessionMemory: 最后回复无Tool调用时触发
- extractMemories: 对话结束时触发
- autoDream: 积累足够会话后触发

#### 4. 三重门控机制

```
时间门控: >= 24h
会话门控: >= 5 sessions
锁门控: 无其他进程整合中
```

#### 5. ForkedAgent异步执行

- 不占用主对话Token
- 共享prompt cache
- 独立Tool权限

#### 6. 记忆时效性

- >1天的记忆自动加陈旧警告
- 强调"快照而非实时状态"

#### 7. 相关性检索

- Sonnet选最多5条
- 精简而非全量加载
