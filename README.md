# 牍典库 | DuDianKu
**积牍为文，汇典成库，铸Agent恒久之思**

专为 AI Agent 打造的**层级化长期记忆 + 结构化知识库**开源框架

---

## 项目简介
牍典库（DuDianKu）是一款极简、可扩展的 Agent 记忆与知识管理框架，以中式古典知识体系为设计理念，构建**「牍 → 典 → 库」**三层递进架构，完美匹配原始数据→知识凝练→体系化存储的全流程，解决 AI Agent 短期记忆遗忘、知识碎片化、无法长效沉淀的核心问题。

框架轻量易用，可无缝集成到各类智能体、大模型应用中，成为 Agent 的**长效知识大脑**。

## 核心理念
恪守「从单元到体系」的知识生长逻辑，三层架构一一对应知识进化全链路：
1. **牍 (Du) - 最小记忆单元**
   原始数据载体，对应 Agent 的单条交互记录、碎片化信息、感知输入、原始文本等最基础的记忆颗粒。
2. **典 (Dian) - 凝练知识范式**
   对「牍」进行自动总结、萃取、去重、归纳，形成标准化的经验、规则、知识点与核心结论，完成从数据到知识的升华。
3. **库 (Ku) - 体系化知识底座**
   将所有「典」结构化存储、索引、检索与复用，构建 Agent 专属的长期知识库，实现知识的持久化与全局共享。

## 架构设计
```
原始数据/交互记忆
        ↓
┌─────────────────┐
│    牍 (Du)      │  最小单元：原始记忆、碎片化数据
└─────────────────┘
        ↓ 自动萃取、编纂
┌─────────────────┐
│    典 (Dian)    │  凝练知识：经验、规则、核心结论
└─────────────────┘
        ↓ 归集、结构化
┌─────────────────┐
│    库 (Ku)      │  知识底座：持久存储、检索复用
└─────────────────┘
```

|---------------------|----------------------------------------------------|

# AI工作原则
1. **工作语言为中文。**
2. **工作目录管理**：
   - 目录请严格按以上目录维护，工作中只能在以上目录中新增修改或删除文件，不在主目录下进行文件处理，如需新增目录，先进行询问，确认后新增。
   - 每次任务按理解和规划，合理管理各目录下的文件增删改，保持良好的项目架构。
   - 为避免过多信息堆积到最上层，代码下每一级目录可以有一个readme，但要按照你的理解合理规划，平衡单文件过于聚集和多文件信息过于分散。
3. **工作知识沉淀**：
   - 不同文档的区别：
      - `module/{project_name}/readme.md`: 项目代码中的readme，主要**面向开发者**，提供使用方法、代码架构、配置信息等开发细节信息，目的是帮助开发者快速、详细了解代码，实现更加高效准确的开发。
      - `doc/{YYYYMMDD}/xxx.md`: 项目的工作信息，主要**面向项目管理**，隐去具体的技术和代码细节，提供项目背景、方案架构、实施规划流程、进度等信息，目的是让不懂技术管理者快速了解当前项目状态，快速参与规划与决策。
      - `wiki/`: 知识库，前面二者最主要的目的都是让相关人员快速了解本项目信息，加速迭代。但知识库是抽象出可跨项目复用的知识，目的是帮助**其他项目**快速了解通用知识、复用经验，以推动整体工作效率的提升。
   - 每次完成工作后，思考：
      - 项目代码改动后，是否有对应更新readme。
      - 是否有信息需要记录到doc，以为后续该任务的继续处理提供细节支持。
      - 是否有核心知识需要记录到wiki中，以为其他任务提供帮助。
